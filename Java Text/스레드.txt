1. 스레드의 개요
OS - 멀티 태스킹
-> 한 대의 컴퓨터로 2가지 이상의 작업을 동시에 처리
-> 2가지 이상의 프로그램들을 동시에 수행 
-> 프로세스 단위가 아닌 Thread 단위로 실행 

* 프로세스와 스레드
- 프로세스 : 메모리를 할당 받아 실행 중인 프로그램
- 프로세스 단위로 작업 처리 시
   -> Memory, CPU 등의 자원 많이 점유
   -> 시스템 과부하 발생
- 스레드 : 프로세스가 일정한 작은 단위로 분할된 것
- 스레드 단위로 작업 처리 시
   -> Memory, CPU 등의 자원 점유율 낮음
   -> 시스템 과부하 해결

< 다중 스레드 > 
- 하나의 프로그램 내에서 다수 개의 스레드가 동시에 수행
- 메모리, CPU 점유율 낮음
- 멀티 태스킹 가능
- 프로그램 수행 속도가 향상
- CPU 효율 향상

* 활용
- Server / Client 구조에서 활용
- JSP / 서블릿 기반 웹 애플리케이션에서 User 1명 -> 1Thread
- 서버 동시 접속자 처리 원할
- 채팅 프로그램 
- 온라인 게임

*) 단일 스레드 VS 다중 스레드
-> 단일 스레드 : 순차적 작업 수행
-> 다중 스레드 : 동시적 작업 수행

* Thread 생성 (2가지)
1. Thread 클래스 상속 (extends)
  - 클래스 전체를 스레드로 사용
  - 체팅 프로그램에서 각 클라이언트를 스레드로 관리
2. Runnable 인터페이스 implements
  - 자바 애니메이션 구현 (JFrame, JApplet)

* Thread 클래스 활용
- run()메소드
- start() 메소드
